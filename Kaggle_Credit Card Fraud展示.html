The data is downloaded from kaggle: credit card fraud detection.

Here I use randomforest to fit the data, and use nested cross validation (Nested CV) to find the best tuning parameter: mtry.

The class is unbalanced, which I need some techniques to handle. I handle it using stratified sampling, but there are other ways to deal with it like oversampling, undersampling, or weighted cost. Here I use SMOTE to deal with this problem.

Step 2. Prepare for Analyzing

Note that when doing nested cross validation, I decide to do the following procedures:

  1. I choose one parameter: mtry (number of splits).

  2. I use averaged F1 score to do evaluation, due to this article: https://stats.stackexchange.com/a/252451. I use F1 score not only because it is intuitive, but also because sometimes AUC may be poor to evaluate models on unbalanced data, as this article says.

library(caret, quietly=TRUE)   #for stratified sampling, computing prec,recall
library(DMwR, quietly=TRUE) #for SMOTE
set.seed(101)
fold <- createFolds(factor(data$Class), k=5, list=FALSE) #Construct five stratified folds

Take fold=1 as an example, the percent of 1 in the fold is equal to one in the whole data set.

x <- model.matrix(Class~.-Time-1, data=data)
# Since "Time" is not meaningful to the analysis, I omit it.
data$Class <- as.factor(data$Class)
y <- data$Class
bestmtry <- rep(0,5)
outerlist <- list()
oconfusionM <- list()
oevaluation <- list()

Step 3. Analyzing: Randomforest and Nested CV

library(randomForest, quietly=TRUE)

for (i in 1:5) {
    F1 <- rep(0,4)
    for (j in 1:4) {
        #inner cv for mtry  
        k <- ifelse(i+j>5, (i+j)%%5, i+j)
        cond <- fold==i | fold==k
        Smote <- SMOTE(Class~.-Time, data=data[cond,], perc.over=300, perc.under=1000)
        xs <- model.matrix(Class~.-Time-1, data=Smote)
        ys <- Smote$Class
        for (m in 2:5) {
            inner <- randomForest(xs, ys, mtry=m)
            inpred <- predict(inner, x[fold==k,])
            determ1 <- factor(inpred, c(1,0))
            determ2 <- factor(y[fold==k], c(1,0))
            inpred.table <- table(determ1, determ2)
            cmatrix <- confusionMatrix(inpred.table, mode="prec_recall")
            F1[m-1] <- F1[m-1] + cmatrix$byClass["F1"]
            }
        }
    maxima <- which(F1==max(F1), arr.ind=TRUE)
    bestmtry[i] <- maxima+1

    oSmote <- SMOTE(Class~.-Time, data=data[fold!=i,], perc.over=300, perc.under=1000)
    xo <- model.matrix(Class~.-Time-1, data=oSmote)
    yo <- oSmote$Class
    outer <- randomForest(xo, yo, mtry=bestmtry[i], importance=TRUE)
    outerlist <- append(outerlist, list(outer))
    pred <- predict(outer, x[fold==i,])
    odeterm1 <- factor(pred, c(1,0))
    odeterm2 <- factor(y[fold==i], c(1,0))
    pred.table <- table(odeterm1, odeterm2)
    outercmatrix <- confusionMatrix(pred.table, mode="prec_recall")
    oconfusionM <- append(oconfusionM, list(outercmatrix$table))
    oevaluation <- append(oevaluation, list(outercmatrix$byClass["F1"]))
    }

Step 4. Check the Consistency of Outer Fold to Determine the Model is Stable or not.

This is great to use nested CV because we can examine the stability of the model. If the model is unstable, it means that when we use different data to fit the same model, the result is not countable, which is problematic.
